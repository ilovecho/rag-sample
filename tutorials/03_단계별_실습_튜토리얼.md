## 실습 1: 임베딩 체험하기

> 목표: 텍스트가 어떻게 숫자로 변환되는지 직접 확인

```python
# 파일명: practice_01_embedding.py
from dotenv import load_dotenv
from langchain_openai import OpenAIEmbeddings
import numpy as np

load_dotenv()

# ── 1. 임베딩 모델 생성 ──
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# ── 2. 하나의 텍스트를 임베딩 ──
text = "고양이는 귀여운 동물이다"
vector = embeddings.embed_query(text)

print(f"원본 텍스트: {text}")
print(f"벡터 차원 수: {len(vector)}")         # 1536
print(f"벡터 앞 5개: {vector[:5]}")           # [0.012, -0.034, ...]
print(f"벡터 타입: {type(vector)}")            # list

# ── 3. 여러 텍스트의 유사도 비교 ──
sentences = [
    "고양이는 귀여운 동물이다",     # 원본
    "냥이는 사랑스러운 반려동물이다", # 의미 유사
    "강아지도 귀여운 동물이다",      # 부분 유사
    "오늘 주식이 폭락했다",          # 관련 없음
]
vectors = embeddings.embed_documents(sentences)

# 코사인 유사도 계산 함수
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print("\n=== 유사도 비교 (원본: '고양이는 귀여운 동물이다') ===")
for i in range(1, len(sentences)):
    sim = cosine_sim(vectors[0], vectors[i])
    bar = "█" * int(sim * 30)
    print(f"  {sim:.4f} {bar} {sentences[i]}")
```

### 실행 결과 예시
```
원본 텍스트: 고양이는 귀여운 동물이다
벡터 차원 수: 1536
벡터 앞 5개: [0.0123, -0.0345, 0.0567, ...]

=== 유사도 비교 (원본: '고양이는 귀여운 동물이다') ===
  0.8921 ██████████████████████████ 냥이는 사랑스러운 반려동물이다
  0.8234 ████████████████████████ 강아지도 귀여운 동물이다
  0.1523 ████ 오늘 주식이 폭락했다
```

### 여기서 배운 것
```
✅ embed_query()  — 검색 질문용 (1개 텍스트)
✅ embed_documents() — 문서 저장용 (여러 텍스트)
✅ 의미가 비슷하면 유사도가 높다
✅ 벡터 차원이 1536개나 된다 (매우 세밀한 의미 표현)
```

## 실습 2: 문서 로드 & 분할

> 목표: 실제 문서를 읽고 적절한 크기로 분할하기

```python
# 파일명: practice_02_split.py
from langchain_community.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

# ── 1. 문서 로드 ──
loader = TextLoader("sample_documents/company_policy.txt", encoding="utf-8")
documents = loader.load()

print(f"문서 수: {len(documents)}")
print(f"전체 길이: {len(documents[0].page_content)}자")
print(f"메타데이터: {documents[0].metadata}")
# → {'source': 'sample_documents/company_policy.txt'}

# ── 2. 분할하지 않았을 때의 문제 ──
print("\n--- 분할 없이 전체 문서 ---")
print(documents[0].page_content[:200])
print("... (이하 생략)")
print("→ 이 전체를 하나의 벡터로 만들면 '연차'도 '식대'도 다 섞임!")

# ── 3. 텍스트 분할 ──
splitter = RecursiveCharacterTextSplitter(
    chunk_size=200,      # 한 청크 최대 200자
    chunk_overlap=30,    # 겹침 30자
    separators=["\n\n", "\n", ". ", " ", ""],
    # ↑ 분할 우선순위: 빈 줄 > 줄바꿈 > 문장 끝 > 공백 > 글자
)

chunks = splitter.split_documents(documents)

print(f"\n--- 분할 결과: {len(chunks)}개 청크 ---")
for i, chunk in enumerate(chunks):
    print(f"\n[청크 {i+1}] ({len(chunk.page_content)}자)")
    print(f"  {chunk.page_content}")
    print(f"  메타데이터: {chunk.metadata}")

# ── 4. chunk_size에 따른 차이 실험 ──
print("\n\n=== chunk_size별 청크 수 비교 ===")
for size in [100, 200, 300, 500]:
    s = RecursiveCharacterTextSplitter(chunk_size=size, chunk_overlap=30)
    c = s.split_documents(documents)
    print(f"  chunk_size={size:>4} → 청크 {len(c):>2}개")
```

### 여기서 배운 것
```
✅ TextLoader로 파일을 Document 객체로 읽음
✅ Document = page_content(내용) + metadata(출처 등)
✅ RecursiveCharacterTextSplitter가 가장 범용적
✅ chunk_size가 작으면 → 청크 많고 정밀, 크면 → 청크 적고 문맥 풍부
✅ separators 순서대로 자연스러운 위치에서 분할
```

## 실습 3: 벡터 저장소 만들기

> 목표: 분할된 청크를 벡터DB에 저장하고 검색하기

```python
# 파일명: practice_03_vectorstore.py
from dotenv import load_dotenv
from langchain_openai import OpenAIEmbeddings
from langchain_community.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import Chroma

load_dotenv()

# ── 1. 문서 준비 ──
loader = TextLoader("sample_documents/company_policy.txt", encoding="utf-8")
documents = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=200, chunk_overlap=30)
chunks = splitter.split_documents(documents)
print(f"준비된 청크: {len(chunks)}개")

# ── 2. 벡터 저장소 생성 ──
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    # persist_directory="./chroma_db",  # 이 줄을 넣으면 디스크에 저장
)
print("벡터 저장소 생성 완료!")

# ── 3. 검색 테스트 ──
print("\n=== 검색 테스트 ===")
queries = ["연차 휴가", "재택근무 규정", "점심 식비"]

for query in queries:
    print(f"\n질문: '{query}'")
    results = vectorstore.similarity_search(query, k=2)
    for i, doc in enumerate(results):
        print(f"  결과{i+1}: {doc.page_content[:80]}...")

# ── 4. 유사도 점수와 함께 검색 ──
print("\n\n=== 유사도 점수 확인 ===")
query = "성과급은 얼마인가요?"
results_with_score = vectorstore.similarity_search_with_score(query, k=3)

for doc, score in results_with_score:
    # Chroma에서 score는 거리(distance) → 낮을수록 유사
    print(f"  거리: {score:.4f} | {doc.page_content[:60]}...")
```

### 여기서 배운 것
```
✅ Chroma.from_documents()로 한 번에 임베딩+저장
✅ similarity_search(query, k=N) — 유사한 문서 N개 검색
✅ similarity_search_with_score() — 점수도 함께 확인
✅ persist_directory를 지정하면 프로그램 종료 후에도 유지
```

## 실습 4: 완성! 질문-답변 RAG

> 목표: 검색 결과를 LLM에 전달하여 자연어 답변 생성

```python
# 파일명: practice_04_qa.py
from dotenv import load_dotenv
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_community.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import Chroma
from langchain.chains import RetrievalQA
from langchain.prompts import PromptTemplate

load_dotenv()

# ── 1~3단계: 문서 준비 (앞서 배운 것) ──
loader = TextLoader("sample_documents/company_policy.txt", encoding="utf-8")
docs = loader.load()
chunks = RecursiveCharacterTextSplitter(
    chunk_size=300, chunk_overlap=50
).split_documents(docs)
vectorstore = Chroma.from_documents(
    chunks, OpenAIEmbeddings(model="text-embedding-3-small")
)

# ── 4단계: LLM 준비 ──
llm = ChatOpenAI(
    model="gpt-4o-mini",   # 모델 선택
    temperature=0,          # 0=일관된 답변, 1=창의적 답변
)

# ── 5단계: 프롬프트 작성 ──
prompt = PromptTemplate(
    input_variables=["context", "question"],
    template="""당신은 회사 정책을 안내하는 AI 어시스턴트입니다.

아래 [참고 문서]만을 근거로 질문에 답변하세요.
문서에 없는 내용은 "해당 정보는 문서에서 확인되지 않습니다"라고 답하세요.

[참고 문서]
{context}

[질문]
{question}

[답변]""",
)

# ── 6단계: RAG 체인 조립 ──
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",    # 검색 결과를 프롬프트에 통째로 넣기
    retriever=vectorstore.as_retriever(search_kwargs={"k": 3}),
    return_source_documents=True,
    chain_type_kwargs={"prompt": prompt},
)

# ── 7단계: 질문하기! ──
questions = [
    "신입사원의 연차는 어떻게 되나요?",
    "재택근무를 하려면 어떻게 해야 하나요?",
    "성과 평가 A등급이면 성과급이 얼마인가요?",
    "주차장 이용 방법을 알려주세요",  # 문서에 없는 내용
]

for q in questions:
    result = qa_chain.invoke({"query": q})
    print(f"\n{'='*50}")
    print(f"Q: {q}")
    print(f"A: {result['result']}")

    # 어떤 문서를 참조했는지 확인
    print(f"\n  📄 참조한 문서:")
    for i, doc in enumerate(result['source_documents']):
        print(f"    [{i+1}] {doc.page_content[:60]}...")
```

### 실행 결과 예시
```
==================================================
Q: 신입사원의 연차는 어떻게 되나요?
A: 입사 1년 미만인 신입사원은 매월 1일씩 연차가 발생합니다.
   1년 이상 근무하시면 연 15일의 연차가 부여됩니다.

  📄 참조한 문서:
    [1] 2. 휴가 정책 - 연차는 입사 1년 미만 시 월 1일씩 발생합니다...
    [2] - 입사 1년 이상부터 연 15일의 연차가 부여됩니다...

==================================================
Q: 주차장 이용 방법을 알려주세요
A: 해당 정보는 문서에서 확인되지 않습니다.
```

### 여기서 배운 것
```
✅ RetrievalQA — 검색+답변을 하나로 묶은 체인
✅ chain_type="stuff" — 가장 단순한 방식 (검색 결과를 통째로 넣기)
✅ return_source_documents=True — 근거 문서를 함께 반환
✅ 커스텀 프롬프트로 답변 형식과 톤을 제어
✅ 문서에 없는 질문에 "모른다"고 답하게 유도 가능
```

## 실습 5: 대화형 RAG (챗봇)

> 목표: "아까 말한 거" 같은 이전 대화를 기억하는 RAG 챗봇

```python
# 파일명: practice_05_chatbot.py
from dotenv import load_dotenv
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_community.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import Chroma
from langchain.memory import ConversationBufferMemory
from langchain.chains import ConversationalRetrievalChain

load_dotenv()

# 문서 준비 (동일)
loader = TextLoader("sample_documents/company_policy.txt", encoding="utf-8")
docs = loader.load()
chunks = RecursiveCharacterTextSplitter(
    chunk_size=300, chunk_overlap=50
).split_documents(docs)
vectorstore = Chroma.from_documents(
    chunks, OpenAIEmbeddings(model="text-embedding-3-small")
)

# ── 핵심: 대화 메모리 ──
memory = ConversationBufferMemory(
    memory_key="chat_history",
    return_messages=True,
    output_key="answer",
)
# → 이전 대화 내용을 자동으로 저장하고 다음 질문에 전달

# ── 대화형 RAG 체인 ──
chain = ConversationalRetrievalChain.from_llm(
    llm=ChatOpenAI(model="gpt-4o-mini", temperature=0),
    retriever=vectorstore.as_retriever(search_kwargs={"k": 3}),
    memory=memory,
    return_source_documents=True,
)

# ── 데모: 문맥을 기억하는 대화 ──
conversations = [
    "휴가 정책에 대해 알려주세요",
    "그 중에서 경조사 휴가는 어떻게 되나요?",     # "그 중에서" ← 이전 맥락!
    "결혼하면 며칠 쉴 수 있나요?",                # 경조사 → 결혼 맥락 유지
]

for q in conversations:
    result = chain.invoke({"question": q})
    print(f"\n사용자: {q}")
    print(f"AI:     {result['answer']}")

# ── 메모리 내용 확인 ──
print("\n\n=== 저장된 대화 히스토리 ===")
for msg in memory.chat_memory.messages:
    role = "사용자" if msg.type == "human" else "AI"
    print(f"  {role}: {msg.content[:80]}...")
```

### 여기서 배운 것
```
✅ ConversationBufferMemory — 대화 기록 저장
✅ ConversationalRetrievalChain — 대화 맥락 + 문서 검색 결합
✅ "그 중에서", "아까 말한" 같은 대명사/참조를 이해함
✅ memory.chat_memory.messages로 저장된 대화 확인 가능
```
