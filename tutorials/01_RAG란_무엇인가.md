# 01. RAG란 무엇인가?

## 한 줄 요약
> **RAG(Retrieval-Augmented Generation)**는 LLM이 답변할 때, 외부 문서에서 관련 정보를 **검색(Retrieval)**해서 함께 참고하도록 하는 기술입니다.

---

## 왜 RAG가 필요한가?

### LLM의 한계

ChatGPT 같은 LLM은 매우 똑똑하지만, 치명적인 한계가 있습니다:

```
❌ 학습 데이터 이후의 정보를 모른다  (2024년 이후 뉴스?)
❌ 우리 회사 내부 문서를 모른다      (사내 정책, 제품 매뉴얼)
❌ 없는 정보를 그럴듯하게 지어낸다    (할루시네이션)
❌ 출처를 알 수 없다                 (어디서 나온 정보인지?)
```

### RAG가 해결하는 것

```
✅ 최신 문서를 검색해서 답변         → 정보가 항상 최신
✅ 우리 회사 문서를 기반으로 답변     → 내부 지식 활용
✅ 문서에 근거한 답변만 생성          → 할루시네이션 감소
✅ 참조한 문서를 함께 보여줌          → 출처 확인 가능
```

---

## RAG vs 파인튜닝 vs 프롬프트 엔지니어링

LLM에 지식을 추가하는 3가지 방법을 비교합니다:

| 구분 | 프롬프트 엔지니어링 | RAG | 파인튜닝 |
|------|---------------------|-----|----------|
| **방법** | 질문할 때 정보를 직접 넣어줌 | 벡터DB에서 검색해서 넣어줌 | 모델 자체를 재학습 |
| **비유** | 시험 볼 때 커닝페이퍼 | 시험 볼 때 교과서 검색 | 교과서를 외우게 함 |
| **비용** | 거의 없음 | 중간 (임베딩 비용) | 높음 (GPU 비용) |
| **최신성** | 수동 업데이트 | 문서만 업데이트하면 됨 | 재학습 필요 |
| **적합한 경우** | 정보가 적을 때 | 대량의 문서 기반 QA | 모델 행동 자체를 바꿀 때 |

> **초보자 추천**: 대부분의 경우 RAG로 시작하는 것이 가장 효율적입니다.

## RAG의 전체 흐름 (Big Picture)

RAG는 크게 **2단계**로 나뉩니다:

### 1단계: 인덱싱 (Indexing) — 사전 준비

문서를 미리 처리해서 검색 가능한 형태로 저장합니다.

```
[원본 문서]          [텍스트 청크]         [벡터(숫자)]        [벡터 DB]
   PDF    ──Load──▶  "연차는..."  ──Embed──▶  [0.1, 0.3, ...]  ──Store──▶  ChromaDB
   TXT               "재택은..."             [0.2, 0.1, ...]               FAISS
   Web                "급여는..."             [0.5, 0.4, ...]               Pinecone
```

### 2단계: 질의응답 (Query) — 실제 사용

사용자가 질문하면 관련 문서를 찾아서 LLM에 함께 전달합니다.

```
[사용자 질문]                    [관련 문서 검색]              [LLM 답변 생성]
"연차 며칠?"  ──Embed──▶ 벡터  ──Search──▶ "연차는 15일..."  ──LLM──▶ "연차는 15일입니다"
                         ↓                  "3년 이상시..."            + 출처 표시
                    [0.1, 0.3, ...]
```

---

## 비유로 이해하기

### RAG = 똑똑한 비서 + 도서관

```
기존 LLM:
  학생: "우리 회사 연차 정책이 뭐야?"
  LLM:  "일반적으로 회사 연차는..." (일반적인 답변, 부정확)

RAG:
  학생: "우리 회사 연차 정책이 뭐야?"
  비서:  (도서관에서 '연차' 관련 문서를 빠르게 찾음)
  비서:  (찾은 문서를 LLM에게 건네줌)
  LLM:  "우리 회사 규정에 따르면, 1년 이상 근속 시 연 15일입니다" ✅
```

## 핵심 구성 요소 한눈에 보기

```
┌─────────────────────────────────────────────┐
│                 RAG 시스템                    │
├─────────────────────────────────────────────┤
│                                             │
│  [Document Loader]  문서를 읽어오는 도구      │
│       ↓                                      │
│  [Text Splitter]    긴 문서를 작은 조각으로    │
│       ↓                                      │
│  [Embedding Model]  텍스트를 숫자 벡터로       │
│       ↓                                      │
│  [Vector Store]     벡터를 저장/검색하는 DB    │
│       ↓                                      │
│  [Retriever]        질문과 유사한 문서 검색    │
│       ↓                                      │
│  [LLM]              검색 결과로 답변 생성      │
│                                             │
└─────────────────────────────────────────────┘
```
