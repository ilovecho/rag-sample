# 04. RAG 용어 사전

## 기본 용어

### LLM (Large Language Model)
```
대규모 언어 모델. ChatGPT, Claude, Gemini 같은 AI 모델.
방대한 텍스트 데이터로 학습되어 자연어를 이해하고 생성할 수 있음.

비유: 수많은 책을 읽은 매우 똑똑한 사람 (하지만 최신 정보는 모름)
```

### RAG (Retrieval-Augmented Generation)
```
검색 증강 생성. LLM이 답변하기 전에 관련 문서를 검색해서 참고하는 기술.

비유: 시험 볼 때 교과서를 찾아보면서 답안을 작성하는 것
```

### 할루시네이션 (Hallucination)
```
LLM이 사실이 아닌 내용을 그럴듯하게 지어내는 현상.
RAG를 사용하면 문서 기반으로 답하므로 할루시네이션이 줄어듦.

예: "한국 최초의 우주비행사는 김철수입니다" (거짓 정보를 자신있게 답변)
```

## 임베딩 관련

### 임베딩 (Embedding)
```
텍스트를 고차원 숫자 벡터로 변환하는 것.
의미가 비슷한 텍스트는 비슷한 벡터가 됨.

"고양이" → [0.82, 0.15, 0.43, ...]  (1536개의 숫자)
```

### 벡터 (Vector)
```
숫자의 배열. 임베딩의 결과물.
[0.1, 0.3, 0.5, ...] 같은 형태.

차원(dimension): 벡터에 포함된 숫자의 개수
  text-embedding-3-small: 1536차원
  text-embedding-3-large: 3072차원
```

### 코사인 유사도 (Cosine Similarity)
```
두 벡터가 얼마나 비슷한지 측정하는 방법 (-1 ~ 1).
  1에 가까움 = 매우 유사
  0에 가까움 = 관련 없음
 -1에 가까움 = 반대 의미

비유: 두 화살표가 같은 방향을 가리키는 정도
```

### 임베딩 모델 (Embedding Model)
```
텍스트를 벡터로 변환하는 AI 모델.

주요 모델:
  OpenAI text-embedding-3-small  — 가성비 좋음 (추천)
  OpenAI text-embedding-3-large  — 더 정밀
  HuggingFace 모델들             — 무료, 로컬 실행 가능
```

## 문서 처리 관련

### 문서 로더 (Document Loader)
```
다양한 형식의 파일을 읽어서 Document 객체로 변환하는 도구.

지원 형식: TXT, PDF, DOCX, CSV, HTML, 웹페이지, Notion, 등
LangChain 예: TextLoader, PyPDFLoader, CSVLoader
```

### Document (문서 객체)
```
LangChain에서 문서를 표현하는 기본 단위.

Document(
    page_content="문서 내용...",    # 텍스트 본문
    metadata={"source": "파일경로", "page": 1}  # 부가 정보
)
```

### 청크 (Chunk)
```
긴 문서를 분할한 작은 텍스트 조각.

왜 분할? → 임베딩 품질 향상 + 정밀한 검색 + LLM 입력 제한
```

### 텍스트 스플리터 (Text Splitter)
```
문서를 청크로 분할하는 도구.

RecursiveCharacterTextSplitter  — 가장 범용적 (추천)
CharacterTextSplitter           — 단순 분할
TokenTextSplitter               — 토큰 기준 분할

주요 파라미터:
  chunk_size    — 청크 최대 크기 (문자 수)
  chunk_overlap — 청크 간 겹치는 크기 (문맥 유지용)
```

### 청크 오버랩 (Chunk Overlap)
```
인접한 청크끼리 겹치는 부분.
문맥이 끊기지 않도록 앞뒤 내용을 조금씩 포함.

[     청크1      ]
            [     청크2      ]
         ↑↑↑
      overlap (겹침)
```

## 벡터 저장소 관련

### 벡터 DB / 벡터 스토어 (Vector Store)
```
벡터를 저장하고 유사도 기반 검색을 제공하는 데이터베이스.

Chroma     — 로컬, 간단, 학습용
FAISS      — 로컬, 고속, 대규모
Pinecone   — 클라우드, 관리형
Weaviate   — 오픈소스, 풍부한 기능
```

### 인덱스 (Index)
```
벡터를 효율적으로 검색할 수 있도록 정리해놓은 구조.
벡터DB 내부에서 자동으로 생성됨.

비유: 도서관의 도서 분류 시스템 (바로 원하는 책을 찾을 수 있게)
```

### 메타데이터 (Metadata)
```
문서에 붙이는 부가 정보. 필터링 검색에 활용.

예: {"source": "policy.pdf", "page": 3, "category": "HR"}
  → "HR 카테고리에서만 검색" 같은 필터링 가능
```

## 검색 관련

### 리트리버 (Retriever)
```
질문과 관련된 문서를 검색하는 도구.
벡터 DB 위에 얹는 검색 인터페이스.

vectorstore.as_retriever(search_kwargs={"k": 3})
→ 유사한 문서 3개를 찾아주는 리트리버
```

### Top-K
```
검색 결과에서 상위 K개만 반환.

k=3 → 가장 유사한 3개 문서만 반환
k가 클수록: 정보 풍부 but 노이즈 증가
k가 작을수록: 정확 but 관련 정보 누락 가능
```

### MMR (Maximal Marginal Relevance)
```
관련성 높으면서도 다양한 내용의 문서를 선택하는 검색 전략.
중복 결과를 줄이고 다양한 관점의 정보를 제공.

파라미터: lambda_mult (0~1)
  0에 가까움 → 다양성 우선
  1에 가까움 → 유사도 우선
```

### 시맨틱 검색 (Semantic Search)
```
의미 기반 검색. 키워드가 아니라 뜻이 통하는 문서를 찾음.

키워드 검색: "연차" → "연차"라는 글자가 포함된 문서만
시맨틱 검색: "연차" → "휴가", "annual leave", "쉬는 날" 등도 검색됨
```

## LLM 체인 관련

### 체인 (Chain)
```
여러 작업을 순서대로 연결한 파이프라인.

예: RetrievalQA = 검색(Retrieval) + 답변(QA) 체인
    질문 → 검색 → 프롬프트 조합 → LLM → 답변
```

### 프롬프트 템플릿 (Prompt Template)
```
LLM에 보낼 프롬프트의 틀.
{변수}를 사용하여 동적으로 내용을 채울 수 있음.

PromptTemplate(
    template="다음을 참고하여 답변: {context}\n질문: {question}",
    input_variables=["context", "question"]
)
```

### chain_type (체인 타입)
```
검색된 문서를 LLM에 전달하는 방식.

"stuff"       — 모든 문서를 프롬프트에 한번에 넣기 (가장 단순, 추천)
"map_reduce"  — 각 문서를 개별 처리 후 결합 (긴 문서용)
"refine"      — 문서를 하나씩 읽으며 답변 점진적 개선
"map_rerank"  — 각 문서로 답변 생성 후 점수 매겨 선택
```

### Temperature (온도)
```
LLM 답변의 창의성/랜덤성 조절 파라미터 (0~2).

0   → 일관되고 사실적인 답변 (RAG에 추천)
0.7 → 적당히 창의적
1+  → 매우 창의적 (사실 기반 답변에는 비추)
```

## 대화형 RAG 관련

### 대화 메모리 (Conversation Memory)
```
이전 대화 내용을 저장하여 맥락을 유지하는 기능.

ConversationBufferMemory  — 모든 대화 저장 (단순, 추천)
ConversationSummaryMemory — 대화 요약 저장 (긴 대화용)
ConversationWindowMemory  — 최근 N턴만 저장
```

### 대화 히스토리 (Chat History)
```
지금까지 주고받은 대화 기록.
이전 맥락을 참조하여 "그것", "아까 말한 거" 등을 이해하게 해줌.
